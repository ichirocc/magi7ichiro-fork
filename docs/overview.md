## 何をするものか
**MAGI** は看護師・スタッフの**勤務表（シフト）を自動で最適化**するツール。現場のシフト管理者が、多数の制約（人員の必要数・連勤ルール・個人の上下限・希望 など）を満たす**配布可能な勤務表**を、手作業より速く作るためのもの。名前は Evangelion の MAGI（MELCHIOR/BALTHASAR/CASPAR）に由来。

設計の軸は **「指1本・最小UI」**：ドラッグに頼らず、主操作は画面下部に固定、違反や希望は色＋形で一目で分かる。

## 主要機能
- **作る**：ボタンひとつで最適化（`onMake`）。下書き（簡易作成）も可。
- **編集**：基本マスター（シフト/群/職員/制約）・希望（ws3）・今月の調整をエディタで編集。
- **分析**：違反の内訳（18種）・俯瞰・チェック概要・ボトルネック・改善提案。
- **配布**：HARD=0 なら「配布可」。CSV / 印刷で書き出し。
- **中断耐性**：最適化は前景サービスで実行。画面を閉じても継続し、中断しても続きから復帰。
- **見方の切替**：勤務表は 7日表示 / カレンダー / 1ヶ月俯瞰、加えて HUD 由来の **集中モード**（円柱フィッシュアイ）。

## フェーズ（作業の流れ）
**探索 → 完成 → 狩猟** の3段で現在地を示す（`OperatorNextActionCard`）：
- **探索**：まだ最適化していない（次の一手＝「作る」）
- **完成**：HARD=0、そのまま配れる（次の一手＝「配布」）
- **狩猟**：必達違反が残る（次の一手＝「違反を退治」）

## 誰のためか
現場の**シフト管理者**。専門知識やドラッグ操作を前提とせず、配布可能な表を素早く得ることを目的にする。
