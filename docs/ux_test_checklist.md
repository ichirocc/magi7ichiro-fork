# 画面UXテスト項目（ユニバーサルデザイン／HID／スマホ特化）

各画面（ホーム・勤務表・編集（月次/年次）・ようす・設定・各ダイアログ）に対して適用する、
**合否判定できる**テスト項目。根拠（規格・原則）を併記。検証手段：実機 文字サイズ200%・
ダーク/UDテーマ・小型/大型/折りたたみ/タブレット・低電力/発熱・機内モード/SAF。
※**読み上げ（TalkBack/スクリーンリーダー）は本プロジェクトの検証対象外**（A4 等は参考掲載）。

凡例: ✅=現状満たす想定 / 🟡=要計測・部分 / ⬜=未対応 / ➖=対象外

## 実機テスト手順（読み上げ対象外・コードで潰せない項目のみ）

`v6-debug-apk`（CIの `v6-engine-check` 成果物）を実機へ導入し、下記を順に実施。各手順は**1つの合否**に対応。

1. **文字サイズ200%リフロー（A5/C13）**：設定→ディスプレイ→フォントサイズを最大。全画面・全ダイアログを開き、
   文言の見切れ／ボタンラベルの2行あふれ／横スクロール発生／重なりが無いことを確認。NG箇所をスクショ。
2. **画面回転・分割・折りたたみ（C7/I2）**：各画面で縦↔横回転、2分割、折りたたみ展開を行い、
   レイアウト崩れ・はみ出し・状態消失が無いこと。最適化中の回転で計算継続・結果反映も確認。
3. **セーフエリア/ジェスチャーナビ（C8）**：ジェスチャーナビ端末で、下端コマンドバーとボトムシート最下段の
   タイルがナビバーに被らず全押下可能か（特に `ShiftPickerSheet` 最終行）。
4. **屋外・低輝度の視認性（②校正）**：輝度を下げ（または屋外想定）、勤務表カレンダーの曜日ヘッダ・罫線、
   実行中グリッドの「今回変化」赤枠（2dp）、違反の色＋形が判別できるか。
5. **5分実行のサーマル/電池（H5/I1/I5）**：最適化を上限まで実行し、発熱で完了するか・画面消灯しないか・
   復帰時に結果が反映されるか。Doze下の完了通知も確認。
6. **プロセスkill復帰（C10/E7/I3）**：実行中に他アプリ多用→killし、再起動で入力復元・途中最良解からの再開提示。
7. **SAF入出力（C14/E1/L3）**：機内モードでJSON/CSVの開く・保存・再取込が往復一致。選択キャンセルで状態不変。

> コードで対応済み（実機では再確認のみ）：インセット（enableEdgeToEdge＋safeDrawing）、動きの抑制（装飾アニメ不在）、
> 画面消灯防止（keepScreenOn）、前景サービス＋スナップショット復帰、タッチ標的48dp、非色手がかり、実測コントラスト。
